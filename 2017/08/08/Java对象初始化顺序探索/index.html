Java对象初始化顺序探索

发表于 2017-08-08 | 分类于 Java基础

摘要：
话不多说，直接上Code。看下面：

我们来分析一下这个执行顺序该是啥样的：
1，首先 为A分配内存空间，然后初始化为A类里的成员变量的默认值（比如int = 0，boolean = false…）和reference类型（Object o = new Object() o就是Object的reference）。此时 a = 0;
2，调用A的构造函数，进入public A() {}中的第一步是调用B的构造函数，tips：java编译器会默认加载类中的空构造函数。
3，初始化B中的成员变量，B类无成员变量，跳过
4，所以此时输出 “B构造函数：0”
5，初始化A类成员变量，将A类成员变量小a赋值100。a = 100;
6，所以此时输出 “A构造函数：0”
7，赋值a = 200。此时a = 200;
8，执行main函数中的输出 “new A():200”

结论是：子类成员变量初始化是在父类构造函数调用完后，在此之前，成员变量的值均是默认值。

tips：在不熟悉原理的情况下，不要在构造函数里插入太多的业务。
