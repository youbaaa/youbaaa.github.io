数据库索引

发表于 2019-04-10 | 分类于 数据库

摘要：数据库索引…

数据库索引的数据库结构基础为B+_Tree，关于B+_Tree详见B与B+树。
数据库索引是用于提高数据库表的访问速度的。

定义：
索引包含键值和逻辑指针（指向逻辑值或者另一个索引），是以B+树为数据结构，每个节点都是一个索引页，由于数据密集，利用索引查找是在I/O上有很大优势。分为聚集索引和非聚集索引。
聚集索引：表数据按照索引的顺序来存储，对于聚集索引，其叶子节点即存储了真实的数据行，不再有另外单独的数据页。一张表种只能创建一个聚集索引，因为真实的数据的物理顺序只能有一种，如果一张表没有聚集索引，那么它称为“堆集（Heap）”。这样的表种没有特定的顺序，所有的新行将被添加到表的末尾位置。
非聚集索引：表数据存储顺序与索引无关。对于非聚集索引，叶结点包含索引字段值以及指向数据页数据行的逻辑指针，该层紧邻数据页，其行数量 与数据表行数量一致。
索引覆盖：SQL中所涉及的字段在复合非聚集索引所包含范围内时，由于非聚集索引的叶子节点包含所有数据行的索引列值，使用这些节点即可返回真正的数据。
--创建聚集索引
create CLUSTERED INDEX 索引名称 ON 表名(字段名)

--创建非聚集索引
create NONCLUSTERED INDEX 索引名称 ON 表名(字段名)

--删除指定约束
alter table 表名
drop constraint 主键约束名称

--将指定字段设置成主键非聚集索引
alter table 表名 
add constraint 主键约束名称 primary key NONCLUSTERED(字段名)  

--创建表指定主键为非聚集索引,默认不写NONCLUSTERED为聚集索引
CREATE TABLE Test
( 
ID INT PRIMARY KEY NONCLUSTERED --非聚集索引
)
